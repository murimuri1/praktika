# 🎮 GameShop - Desktop Application (WPF)

📌 Описание проекта

GameShop — это десктопное приложение для цифровой дистрибуции видеоигр, созданное по образу и подобию платформы Steam. Проект разработан в рамках учебной практики/курсовой работы и демонстрирует навыки проектирования архитектуры, работы с реляционными базами данных и создания современного пользовательского интерфейса.

🛠 Технологический стек

Язык программирования: C#

Платформа: .NET Framework / WPF (Windows Presentation Foundation)

База данных: MS SQL Server

ORM: Entity Framework (Database First / Code First)

Архитектурные паттерны: Code-Behind, Data Binding, Value Converters.

🚀 Основной функционал (Техническое задание)

1. Система авторизации и ролей

Вход по логину и паролю.

Альтернативный вход (QR-код).

Разделение прав доступа:

User (Пользователь): может просматривать каталог, покупать игры, просматривать свою библиотеку.

Admin (Администратор): имеет расширенные права, включая кнопки «Правка» и «Удалить» для управления ассортиментом магазина.

2. Каталог игр (Магазин)

Отображение игр в виде адаптивных карточек.

Поиск: Фильтрация списка игр в реальном времени по названию.

Сортировка: По умолчанию, по возрастанию цены, по убыванию цены.

Динамический рейтинг: Оценка игры конвертируется в визуальные звезды (★) с помощью кастомных IValueConverter. Цветовая дифференциация:

🔴 1-2 звезды (Красный)

🟡 3 звезды (Желтый)

🟢 4-5 звезд (Зеленый)

Подробная информация: Жанр, разработчик, цена, описание.

3. Библиотека пользователя

Отображение только тех игр, которые привязаны к аккаунту текущего авторизованного пользователя (UserGame).

Счетчик купленных игр с правильным склонением слов (игра/игры/игр).

Кнопка «ИГРАТЬ» для симуляции запуска приложения.

4. Модуль оплаты (Checkout)

Специализированная форма для проведения транзакции.

Валидация полей ввода:

Номер карты (только цифры, 16 символов).

Срок действия (форматирование ММ/ГГ в реальном времени, только цифры).

CVV код (только цифры, скрытый ввод PasswordBox, 3 символа).

Предотвращение повторной покупки игры (проверка IsOwned).

🗄 Структура базы данных

Реляционная БД состоит из следующих основных сущностей:

[User] — данные пользователей.

Role — роли (Admin/User).

Game — каталог игр с привязкой к разработчикам, издателям и жанрам.

Genre, Developer, Publisher — справочники.

UserGame — таблица связи "Многие-ко-многим", реализующая функционал библиотеки.

Review — отзывы и оценки игр пользователями.

💻 Установка и запуск

Клонируйте репозиторий с помощью git clone

Выполните SQL скрипт script.sql в вашем SQL Server Management Studio (SSMS) для создания базы данных и заполнения её тестовыми данными.

Откройте файл решения .sln в Visual Studio.

В файле конфигурации (App.config) обновите строку подключения connectionString под ваш локальный сервер SQL.

Обновите модель Entity Framework (откройте .edmx файл -> Обновить модель из базы данных).

Запустите проект (F5).

Тестовые аккаунты:

Админ: Логин admin, Пароль 123

Пользователь: Логин user, Пароль 123
